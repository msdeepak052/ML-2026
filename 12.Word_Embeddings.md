# 1. What Problem Do Word Embeddings Solve?

Earlier NLP methods represented words like this:

| Word  | Representation |
| ----- | -------------- |
| King  | [0,0,0,1,0]    |
| Queen | [0,0,1,0,0]    |
| Apple | [1,0,0,0,0]    |

These are **One-Hot Encodings**.

### Problems

1. Huge vectors if vocabulary is large.
2. No semantic meaning.
3. Words with similar meaning are **not close**.

Example:

```
King = [0,0,0,1,0]
Queen = [0,0,1,0,0]
```

Distance between them = large
But **semantically they are similar**.

---

# 2. Intuition of Word Embeddings

Word Embeddings represent words as **dense vectors containing meaning**.

Example:

| Word  | Vector             |
| ----- | ------------------ |
| King  | [0.75, 0.12, 0.98] |
| Queen | [0.72, 0.15, 0.95] |
| Apple | [0.10, 0.88, 0.15] |

Now:

```
King ≈ Queen
```

Because their vectors are **similar**.

So embeddings capture **semantic relationships between words**.

---

Example:

```
          King

        Prince
```

Another cluster:

# 4. Famous Word Embedding Example

One famous relationship:

```
King - Man + Woman ≈ Queen
```

Vectors capture **relationships between words**.

Example:

```
Paris - France + Italy ≈ Rome
```

This is why embeddings are powerful.

# 5. How Word Embeddings Work (Concept)

Word embeddings are learned from **context**.

Example sentence:

```
The cat sits on the mat
```

The model learns:

* cat appears near

  * sits
  * mat

So the model understands:

```
cat ≈ animal
dog ≈ animal
```

because they appear in similar contexts.

This concept is called:

### Distributional Hypothesis

> Words appearing in similar contexts have similar meanings.

# 6. Types of Word Embeddings

### 1️⃣ Word2Vec

Developed by Google (2013)

Two architectures:

* CBOW (Continuous Bag of Words)
* Skip-Gram

---

### 2️⃣ GloVe

Developed by Stanford.

Uses **global word co-occurrence statistics**.

---

### 3️⃣ FastText

Developed by Facebook.

Uses **subwords**, good for rare words.

---

### 4️⃣ Contextual Embeddings (Modern)

Examples:

* BERT
* GPT
* ELMo

Here embeddings **change based on sentence context**.

Example:

```
bank (river bank)
bank (financial bank)
```

Different vectors.

# 8. Python Example (Using Gensim Word2Vec)

Install library

```python
pip install gensim
```

---

### Example Dataset

```python
from gensim.models import Word2Vec

sentences = [
    ["king","queen","man","woman"],
    ["paris","france","rome","italy"],
    ["apple","orange","banana","fruit"],
    ["car","bus","truck","vehicle"]
]
```

---

### Train Word2Vec Model

```python
model = Word2Vec(sentences, vector_size=5, window=2, min_count=1)
```

Explanation

```
vector_size = embedding dimension
window = context window size
min_count = minimum word frequency
```

---

### Get Word Vector

```python
print(model.wv["king"])
```

Output example:

```
[0.34, -0.21, 0.56, 0.11, 0.45]
```

---

### Find Similar Words

```python
model.wv.most_similar("king")
```

Example Output:

```
[('queen', 0.89), ('man', 0.72)]
```

---

### Famous Analogy

```python
model.wv.most_similar(positive=["king","woman"], negative=["man"])
```

Expected result

```
queen
```

# 9. Visualizing Word Embeddings

Using PCA:

```python
from sklearn.decomposition import PCA
import matplotlib.pyplot as plt

words = list(model.wv.index_to_key)
vectors = model.wv[words]

pca = PCA(n_components=2)
result = pca.fit_transform(vectors)

plt.scatter(result[:,0], result[:,1])

for i, word in enumerate(words):
    plt.annotate(word, xy=(result[i,0], result[i,1]))

plt.show()
```

This shows **word clusters**.

# 10. Advantages of Word Embeddings

### 1️⃣ Captures semantic meaning

```
King ≈ Queen
Apple ≈ Orange
```

---

### 2️⃣ Dense representation

Instead of:

```
10000 dimensional one-hot vector
```

We use:

```
100 or 300 dimensional embedding
```

---

### 3️⃣ Works better for ML models

Algorithms perform better because embeddings capture meaning.

---

### 4️⃣ Can capture relationships

Example:

```
Paris - France + Italy ≈ Rome
```

---

# 11. Disadvantages of Word Embeddings

### 1️⃣ Context independent (traditional embeddings)

Example:

```
bank
```

Meaning could be:

* river bank
* financial bank

But Word2Vec gives **one vector only**.

---

### 2️⃣ Needs large training data

Good embeddings require **millions of sentences**.

---

### 3️⃣ Cannot capture sentence meaning

Word embeddings represent **words**, not sentences.
