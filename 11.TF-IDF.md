## TF-IDF (Term Frequency – Inverse Document Frequency)

TF-IDF is a technique used in **NLP to measure how important a word is in a document relative to a collection of documents (corpus).**

It improves **Bag of Words** by giving **higher weight to important words** and **lower weight to very common words**.

---

# 1. Why TF-IDF is Needed

Problem with **Bag of Words**:
It only **counts words**, but does not measure **importance**.

Example corpus:

```text
Doc1: The cat sat on the mat
Doc2: The dog sat on the sofa
Doc3: The cat chased the mouse
```

Words like:

```
the, on, sat
```

appear in **almost every document**, so they are **not useful for distinguishing documents**.

TF-IDF **reduces the weight of such common words**.

---

# 2. Core Intuition

TF-IDF =

```
How often the word appears in the document
        ×
How rare the word is across all documents
```

So:

```
Important word = High TF + Low DF
```

Where:

* **TF** → Term Frequency
* **DF** → Document Frequency

---

# 3. Term Frequency (TF)

Measures **how often a word appears in a document**.

Formula:

```
TF(word) = (Number of times word appears in document)
           -----------------------------------------
           (Total words in document)
```

Example document:

```
Doc: NLP is fun and NLP is powerful
```

Word counts:

| Word     | Count |
| -------- | ----- |
| NLP      | 2     |
| is       | 2     |
| fun      | 1     |
| and      | 1     |
| powerful | 1     |

Total words = **7**

Example TF for **NLP**

```
TF(NLP) = 2 / 7 = 0.28
```

---

# 4. Inverse Document Frequency (IDF)

Measures **how rare a word is across documents**.

Formula:

```
IDF(word) = log ( Total Documents / Documents containing the word )
```

Example corpus:

```
Doc1: I love NLP
Doc2: NLP is amazing
Doc3: I love machine learning
```

Word **NLP**

Appears in:

```
Doc1
Doc2
```

So:

```
Total documents = 3
Documents containing NLP = 2
```

IDF:

```
IDF(NLP) = log(3 / 2)
```

Rare words get **higher IDF values**.

---

# 5. TF-IDF Calculation Example

Corpus:

```
Doc1: I love NLP
Doc2: I love machine learning
Doc3: NLP is fun
```

Vocabulary:

```
I, love, NLP, machine, learning, is, fun
```

Now compute TF-IDF.

Example word:

```
NLP in Doc1
```

TF:

```
1 / 3
```

IDF:

```
log(3/2)
```

TF-IDF:

```
TF × IDF
```

This gives **importance of the word in that document**.

---

# 6. Intuitive Understanding

Example dataset:

```
Doc1: Apple releases new iPhone
Doc2: Apple launches new Macbook
Doc3: Samsung releases new phone
```

Common words:

```
new
releases
```

Rare words:

```
iphone
macbook
samsung
```

TF-IDF assigns:

| Word    | Importance |
| ------- | ---------- |
| iphone  | High       |
| macbook | High       |
| samsung | High       |
| new     | Low        |

This helps models understand **document topics**.

---

# 7. Visual Intuition

Bag of Words for a sentence:

```
machine learning is amazing
```

BoW vector:

```
[1,1,1,1]
```

TF-IDF vector might become:

```
[0.8, 0.9, 0.1, 0.7]
```

Where **common words get lower scores**.

# 8. Python Implementation (Scikit-Learn)

```python
from sklearn.feature_extraction.text import TfidfVectorizer

corpus = [
    "I love NLP",
    "I love machine learning",
    "NLP is amazing"
]

vectorizer = TfidfVectorizer()

X = vectorizer.fit_transform(corpus)

print("Vocabulary:", vectorizer.get_feature_names_out())
print("TF-IDF Matrix:\n", X.toarray())
```

---

# 9. Code Explanation

### TfidfVectorizer()

Automatically performs:

1. Tokenization
2. Vocabulary creation
3. TF calculation
4. IDF calculation
5. TF-IDF weighting

---

### Output Example

Vocabulary:

```
['amazing','learning','love','machine','nlp']
```

Matrix:

```
[[0.0,0.0,0.7,0.0,0.7]
 [0.0,0.7,0.5,0.7,0.0]
 [0.7,0.0,0.0,0.0,0.7]]
```

Each value represents **importance of a word in a document**.

# 10. Advantages of TF-IDF

### 1️⃣ Identifies Important Words

Example:

```
Doc1: Python is great for data science
Doc2: Data science uses Python
```

TF-IDF highlights:

```
python
data
science
```

as important keywords.

---

### 2️⃣ Reduces Impact of Common Words

Example words:

```
the
is
and
```

These get **very low weights**.

---

### 3️⃣ Improves Text Classification

Used in:

* Spam detection
* Document classification
* Topic detection

Example spam:

```
free offer
win money
limited deal
```

TF-IDF highlights **rare but meaningful phrases**.

---

### 4️⃣ Works Well with Classical ML

Common models:

* Logistic Regression
* SVM
* Naive Bayes

---

# 11. Disadvantages of TF-IDF

### 1️⃣ Still Ignores Word Order

Example:

```
dog bites man
man bites dog
```

TF-IDF treats them **almost the same**.

---

### 2️⃣ Cannot Capture Context

Example:

```
good
very good
not good
```

TF-IDF does not understand **negation properly**.

---

### 3️⃣ High Dimensionality

If vocabulary has:

```
50,000 words
```

Then TF-IDF vector size = **50,000 features**.

---

### 4️⃣ No Semantic Meaning

Example:

```
car
automobile
vehicle
```

TF-IDF treats them as **completely different words**.

---

# 15. Important Interview Questions

**Q1: What problem does TF-IDF solve?**

It reduces the importance of **very common words** and highlights **rare informative words**.

---

**Q2: Why use log in IDF?**

To **smooth the effect of rare words** and avoid extremely large values.

---

**Q3: What comes after TF-IDF in modern NLP?**

Word embeddings:

* **Word2Vec**
* **GloVe**
* **FastText**
* **BERT embeddings**
